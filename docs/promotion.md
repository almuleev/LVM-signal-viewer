# Promotion Plan

Short, repeatable steps to increase stars, feedback, and downloads.

## 1) Improve Repository Surface

- Keep README product-first (download link, formats, features, controls).
- Add real screenshot and short GIF under `docs/assets/`.
- Keep changelog active and specific.
- Use focused GitHub topics (see `docs/github-topics.md`).

## 2) Release Cadence

- Publish small but regular releases.
- Include concise release notes:
  - what changed
  - why it matters
  - migration or compatibility notes

## 3) Feedback Loops

- Pin a GitHub issue for feedback templates.
- Ask users to attach anonymized sample files for parser bugs.
- Label issues clearly: `bug`, `format`, `performance`, `enhancement`.

## 4) Share Channels

- LabVIEW / instrumentation forums
- Python desktop app communities
- LinkedIn / X short release announcements
- Relevant subreddits (when allowed by rules)

## 5) Reusable Announcement Template

```
LVM Signal Viewer <version> is out.
- <feature 1>
- <feature 2>
- <fix>
Download: <release link>
Feedback and sample files are welcome.
```

## 6) Credibility Signals

- Keep CI green.
- Include tests for parser edge cases.
- Avoid claiming unsupported formats.
- Respond quickly to bug reports with reproducible steps.
